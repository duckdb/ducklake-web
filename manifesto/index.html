The DuckLake manifesto

Simplicity and efficiency

Data lakes are important building blocks in modern architectures. They must be easy to use and performance-oriented by design.

Using a database system for metadata

Data lakes require a catalog. Why not leverage a robust database system for this, capable of handling richer metadata too?

SQL and Parquet readability

To ensure portability and the longevity of the data, your data lake should be accessible via standard SQL queries and Parquet tools.
